# MewCode 版本命令规格

## 背景

MewCode 当前没有供用户查询程序版本的斜杠命令。开发者安装不同构建后，无法在会话内快速确认正在运行的版本。

## 目标用户

- 本地开发、安装和调试 MewCode 的开发者。
- 需要在问题报告中确认 MewCode 版本的用户。

## 能力清单

- 用户可通过版本命令查看当前 MewCode 版本。
- 本地未注入版本信息的开发构建具有明确的开发版本标识。
- 发布构建可在编译阶段注入版本信息，无需修改源码。
- 版本命令出现在统一命令帮助和补全面板中。

## 非功能要求

- 遵循现有集中式斜杠命令注册和分发机制。
- 版本信息由独立模块维护，避免与命令处理或启动流程耦合。
- 命令执行不访问网络、Git 仓库或文件系统。
- 新增行为由自动化测试覆盖，完整 Go 测试无回归。

## 设计骨架

- 独立版本模块保存构建版本并提供稳定的展示结果。
- 命令注册中心增加一个本地版本查询命令。
- 构建工具通过 Go linker flags 覆盖版本模块中的开发默认值。
- 使用文档说明命令用法和发布构建的注入方式。

## 不做范围

- 不显示 Git commit、构建时间、Go 版本、操作系统或 CPU 架构。
- 不在运行时读取 Git tag、环境变量或远程版本信息。
- 不提供版本检查、升级提示或自动更新。
- 不增加版本命令的短别名。

## 完成定义

见 [checklist.md](checklist.md)。全部验收项通过即认为功能完成。
