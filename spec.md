# MewCode Claude Code 风格全屏 TUI 规格

## 背景

MewCode 已具备全屏 TUI、结构化 transcript、流式 Agent 事件、命令补全、权限确认和状态栏，但当前界面仍偏向字符串日志与状态面板：视觉元素密度高，用户、助手、工具和系统信息的层级不够自然，长任务输出容易形成日志墙，命令与权限交互也缺少明确的当前选择。

本章以 Claude Code 的轻量单列交互为参考，在不改变 Agent 行为的前提下重新组织全屏 TUI 的视觉层级与交互反馈，使用户能快速判断 Agent 正在做什么、已经完成什么、何时需要自己操作。

## 目标用户

- 长时间在终端中使用 MewCode 完成编码任务的开发者。
- 需要观察 Agent、工具与权限状态，但不希望被底层日志淹没的用户。
- 使用中文输入、窄终端或频繁调用斜杠命令的用户。

## 能力清单

- 全屏界面采用内容优先的轻量单列布局，减少大面积底色、边框和重复标签。
- 欢迎块始终保留在 transcript 顶部，并与后续对话保持一致的视觉语言。
- 用户、助手、thinking、工具、权限、usage、错误和系统消息具有清晰且克制的层级。
- 助手正文继续以单个 block 流式更新，过程状态不会切碎最终回复。
- thinking 只展示动态阶段和耗时，不展示原始 thinking 内容。
- 工具成功结果默认折叠为名称、目标、状态和耗时；失败、拒绝与拦截自动展开原因。
- 用户主动浏览历史时暂停自动跟随，新输出到达后提供返回底部提示。
- 命令面板支持实时过滤、键盘选择、补全与执行。
- 权限确认以输入区上方的多行面板呈现，选择后在 transcript 留下简洁决策记录。
- header、输入区和状态行根据终端宽度自适应，优先保留当前任务最有价值的信息。

## 非功能要求

- 不改变 Agent Loop、Provider、工具执行和权限决策语义。
- 不把纯 UI activity、命令候选或权限面板状态写入聊天历史和会话存档。
- 保持中文输入、光标编辑、Tab 补全、busy 输入锁和权限快捷键可用。
- 全屏刷新不得为每个 token 创建独立 transcript block。
- 颜色需在深色终端中有足够对比度，并在无颜色或窄终端下保持可读。
- fallback 行式 TUI 保持现有交互和输出行为。
- TUI 内部职责保持清晰，渲染、面板、transcript 与状态分发可独立测试。

## 设计骨架

- 界面由极简 header、滚动 transcript、上下文面板、输入或 activity 区、细状态行组成。
- transcript 使用结构化 block 保存语义状态，由统一主题映射为符号、颜色、缩进和文本。
- 每轮 Agent 输出中的正文、thinking、工具与 usage 由轮次关系组织，而不是简单追加日志行。
- 工具 block 支持运行中、成功、失败、拒绝和拦截状态，并按结果严重程度决定展开程度。
- 命令面板维护独立的候选集合和选中位置，不改变命令注册中心的执行职责。
- 权限面板复用既有权限请求和决策类型，完成后只生成 UI 决策记录。
- viewport 维护自动跟随状态，区分“用户正在查看历史”和“应跟随最新输出”。
- 主题与布局集中定义，使宽屏、标准终端和窄屏遵循同一信息优先级。

## 不做范围

- 不展示或持久化模型原始 thinking 内容。
- 不新增多行输入、输入历史搜索或鼠标交互。
- 不修改命令业务逻辑、权限模式或永久授权语义。
- 不新增 diff 查看器、文件树、侧边栏或多窗格布局。
- 不重写 fallback 行式 TUI。
- 不要求 Agent 生成额外的阶段化结构数据。
