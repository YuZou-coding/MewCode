# MewCode MCP 延迟发现规格

## 背景

MewCode 当前启动时连接全部 MCP server 并注册所有远端工具。这会拖慢启动，并在多数任务不需要 MCP 时扩大模型工具上下文。

## 目标用户

- 配置多个 MCP server，但只在部分任务中使用它们的开发者。
- 希望终端 Agent 快速启动，并按任务需要加载外部能力的用户。

## 能力清单

- 启动时只加载 MCP server 配置，不创建外部连接或发现远端工具。
- 模型通过内置工具按关键词搜索 MCP server、工具名称和能力描述。
- 模型可通过本地工具名称精确加载单个 MCP 工具。
- 精确加载只连接名称对应的 server，不回退到全量发现。
- 搜索成功后，匹配的远端工具加入后续模型请求的可用工具集合。
- 重复搜索复用已建立连接和已注册工具。
- 单个 server 发现失败不会阻断其他 server 的搜索和使用。
- TUI 持续显示已成功连接的 MCP server 数量。

## 非功能要求

- 行为参考 Claude Code 的 deferred tools 模式，只在需要时扩大工具上下文。
- 远端工具继续使用现有权限检查、事件输出和结果回灌流程。
- 不改变用户级与项目级 server 配置合并和覆盖规则。
- 延迟发现、缓存和失败隔离由自动化测试覆盖。

## 设计骨架

- 外部管理器维护 server 配置和已连接 client 缓存。
- 内置搜索工具负责选择候选 server、发现工具、过滤匹配项并注册远端执行器。
- 普通关键词搜索查询尚无本地索引的候选 server，已连接 server 复用发现结果。
- 精确选择先从本地工具名称解析 server，只查询对应 server。
- Agent 每轮从注册中心重新取得工具定义，因此下一轮能看到新注册的远端工具。
- 搜索结果分别返回匹配工具和逐 server 错误。

## 不做范围

- 不在启动阶段预热 MCP 连接。
- 不持久化发现状态到磁盘。
- 不新增用户手动发现命令。
- 不建立独立的持久化 MCP 工具索引。
